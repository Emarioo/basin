
Descriptions for markers that exist in the code.

|Mark|Description|
|-|-|
|`@NOTE`|            exists to emphasize that it is a note. Normal comments are technically notes too.|
|`@NOCHECKIN`|       Things that must be fixed soon, preferably before commiting and pushing.|
|`@TODO`|            Things that will improve the program. Not urgent todos.|
|`@IMPORTANT`|       Something very important in the place it is located. When writing code in that place you must read this or you may assume something that is incorrect.|
|`@OPTIMIZE`|        places to optimize compile time.|
|`@OPTIMIZE_OUTPUT`| places to optimize code generation.|
